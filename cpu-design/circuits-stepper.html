The Stepper Circuit: Cycling Control Signals

Introduction to the Stepper

The **Stepper** circuit is essential for controlling the sequence of operations in the CPU. It cycles through the set-enable steps required for writing and reading data onto and off the bus. Based on our design, we need the stepper to cycle through **six distinct steps repeatedly**.

The Basic Unit

The basic unit for the stepper is two **S-R Latches** (1-bit memory) connected in series. This configuration allows the output of the second latch to be set to '1' only when the value set in the first latch is '1' and the set signal for the second latch is '1'. Otherwise, the output is '0'.

You should start by building this setup and running a simulation to observe its function. This mechanism allows the input value of '1' to be passed down the chain, step-by-step, thus energizing the outputs in a controlled sequence.

The Final Stepper and Looping

To achieve the required control flow, we must **cascade or chain** a series of these basic units together. This allows us to cycle through several outputs, ensuring only a single output is logic 'HI' (or '1') at any given time.

Ultimately, we set up our stepper to have **six distinct outputs** and configure it to loop continuously through those six steps. The video below demonstrates the final circuit setup.
