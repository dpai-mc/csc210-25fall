LOAD, MARS, and END

On our CPU, we need to set inputs for:

  • LOAD - will allow us to load a program into memory
  • MARS - will allow us to manually set the Memory Address Register

We also need to set an output pin for when our program finishes processing, called END

  • LOAD - a single bit input that connects to the "LOAD" pin in our final Control Section. This pin also has a tunnel named "LOAD" on it that is connected to the RAM Input MUX.
  • MARS - is a single bit input that connects to the MAR_S output pin of our Control Section, through an "OR" gate, that allows us to manually set the RAM Memory Address Register
  • END - is a single bit output that connects to the END output from our Control Section. Additionally, this output has a Tunnel on it (END) that connects to the Clock input to stop our CPU from processing once the program END is reached.
