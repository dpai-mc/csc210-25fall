Counter 256 Circuit

The last circuit/modification is to add a counter that counts from 0 to 255 (Hex 00 - FF). We will use this to cycle through our 256 bytes of RAM and load the appropriate assembly language command to run our program.

The main circuit, "256," has a "LOAD" input from the "LOAD" pin via a tunnel (Circuits: Final CPU: LOAD, MARS, END)

This "LOAD" input is ANDed with the "clkl" input from our "Clock" via a tunnel (Circuits: Final CPU - Clock Input)

The "LOAD" input is also used as the enable on a buffered output attached to the output side of the "clk 256" circuit.

The "256" circuit uses a constant input and the "clk" input based upon a "counter" memory circuit (discussed below). There are also 2 outputs:

A 1-bit carry output that is NOTted (we will not use this) and an 8-bit output we call "count." The counter basically counts from 0 to 255 at each clock pulse and we can use this as a RAM memory address to load our programs.

The counter used in the circuit is found in the "Memory" folder and is called "Counter." We need to modify the behavior by setting the following 2 behaviors:

  • Action On Overflow - Set to "Stay at value"
  • Trigger - "Falling Edge"

The default for all of the other options should match those shown in the figure below. If not, then set them to the values in this provided image.

The circuit can then be redrawn where:

  • c - is the carry output pin that we will not use
  • clk - is the ANDed LOAD, clkl input
  • output pin (on the right) will be the 8-bit output labeled "count" in the circuit details provided 2 images above.
