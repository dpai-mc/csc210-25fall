What's a Decoder?

What's a Decoder? A decoder is a form of multiplexor ... Based upon a given set of inputs, we get one and only 1 output from each combination. For example, a 2x4 decoder has 2 inputs and 4 possible outputs. Constructing a 2×4 Decoder

To create this simple decoder, we start with 2 simple 1-bit inputs, labeled A and B. We then run both inputs through NOT gates, to get "Not A" and "Not B." By effectively splitting A and B so that we have 4 usable inputs, A, B, Not A and Not B, we then combine these inputs, as appropriate, into 4, 2 input, AND gates to achieve out 4 possible outputs. If you build the circuit below in Logisim and run the simulation, you can test and verify that when you set A and B both to 0, Output 0 is 1. When you set A to 0 and B to 1, Output 1 is 1 and all others are 0. You can do this to verify that for each of the possible input combinations for A and B, only one of the outputs is 1 and all others are 0.

Extending to a 3×8 Decoder

In your final design, you will need a 3x8 decoder that is part of the ALU and some 4x16 decoders that are part of RAM (how you select individual RAM bytes by row and column). To create a 3x8 decoder, you need to extend the concept above for a 2x4 decoder. In a 3x8, you have 3 inputs, A, B, & C. Each input will then be split and NOTted for a total of 6 possible values (similar to above where you have 2 inputs, A & B, and each in NOTted to create 4 possible values). The 8 outputs come from have 8 possible input sets with 3 inputs like 0,0,0 and 0,0,1 or 1,0,1. So for outputs, you will have 8 outputs made from 3 input AND gates where each AND gate has one input each from A or Not A, B or Not B, and C or Not C.

Extending to a 4×16 Decoder

The 4x16 decoder, is one more extrapolation to 4 inputs, A, B, C, & D. Each NOTted for a total of 8 values. With 4 bits, you have 16 possible combinations so your 16 outputs are 16 AND gates each with 4 inputs.
