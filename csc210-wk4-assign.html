Part 1: 1-Bit Memory Circuit (NAND Gate S-R Latch)

Create a fundamental 1-bit memory circuit using NAND gates to implement an S-R (Set-Reset) latch. This circuit forms the basic building block for all computer memory systems.

Technical Requirements

  • Input pins: Set (S), Reset (R), and Enable (optional)
  • Output pins: Q (stored bit) and QÌ„ (complement)
  • Implementation: Use only NAND gates from Logisim's library
  • Behavior: Must maintain stored state when inputs are inactive
  • Integration: Add to your existing .circ file containing the 8-bit adder

Understanding S-R Latch Operation

The S-R latch is a bistable circuit that can store one bit of information. When the Set input is activated, the output Q becomes 1. When Reset is activated, Q becomes 0. The key feature is that the circuit "remembers" its state even after the input signals are removed.

NAND Gate Implementation Benefits

Using NAND gates provides several advantages: they are universal gates (can implement any logic function), are typically faster than other gate types, and closely match how real computer memory is implemented in hardware.

Testing Your Circuit

  • Test all input combinations: 00, 01, 10, 11
  • Verify that the circuit maintains its state when both inputs are 0
  • Confirm that the forbidden state (both inputs = 1) behaves predictably
  • Use Logisim's simulation tools to observe the timing behavior

Part 2: 8-Bit Register

Expand your 1-bit memory circuit to create an 8-bit register capable of storing a full byte of data. This register will use eight instances of your 1-bit memory circuit with shared control signals.

Technical Requirements

  • Capacity: Store 8 bits of data simultaneously
  • Input: 8-bit data input bus + control signals (Enable/Clock)
  • Output: 8-bit data output bus
  • Control: Shared enable signal for all 8 bits
  • Components: Built using eight of your 1-bit memory circuits
  • Organization: Clear, organized layout in Logisim

Register Architecture

An 8-bit register consists of 8 individual 1-bit storage elements (your memory circuits) that operate in parallel. All bits share common control signals, allowing the entire byte to be loaded or read simultaneously.

Control Signal Design

Implement a shared enable signal that controls when data is loaded into the register. Consider how this signal should be distributed to all 8 memory circuits and how it affects the Set and Reset inputs of each latch.

Data Bus Organization

Organize your inputs and outputs as 8-bit buses in Logisim. Use consistent naming conventions (e.g., D0-D7 for inputs, Q0-Q7 for outputs) and maintain a clean, readable layout.

Testing Strategy

  • Test loading different 8-bit values (00000000, 11111111, 10101010, etc.)
  • Verify that data persists when the enable signal is inactive
  • Test individual bit operations if your design supports them
  • Use Logisim's probe tool to monitor all 8 outputs simultaneously
