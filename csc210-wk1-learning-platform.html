Welcome to Computer Architecture

This week establishes the foundation for understanding how computers work at the hardware level. You'll explore the layers of abstraction that make modern computing possible and master the number systems that computers use internally.

What You'll Learn This Week

  • The relationship between hardware and software through abstraction layers
  • How to convert numbers between decimal, binary, and hexadecimal systems
  • Signed integer representation using two's complement notation
  • How data is stored and manipulated at the bit level
  • Major components of computer systems and their roles

The Big Picture

By the end of this course, you'll build a complete 8-bit CPU from scratch! This week's number systems form the foundation for everything else we'll learn. Understanding binary and hexadecimal isn't just academic - it's essential for:

  • Debugging low-level programs
  • Understanding memory addresses
  • Working with embedded systems
  • Optimizing software performance
  • Digital circuit design

Learning Objectives

By the end of this week, you will be able to:

  • Explain the relationship between hardware and software through abstraction layers
  • Convert numbers between decimal, binary, and hexadecimal representations
  • Represent signed integers using two's complement notation
  • Describe how data is stored and manipulated at the bit level
  • Identify the major components of a computer system and their roles

How These Align with Course Goals

These objectives directly support the course's main goals:

  • Write assembly programs: Number systems are essential for understanding memory addresses and data representation
  • Describe computer architecture: Abstraction layers show how components work together
  • Understand implementation: Binary representation is how high-level constructs are actually stored
