Course Description

This hands-on laboratory course provides practical experience with computer organization and architecture concepts. Students will design and build digital circuits using Logisim simulation software, culminating in the construction of a complete 8-bit CPU. The course includes extensive practice with MIPS assembly language programming using the MARS simulator. Lab activities are synchronized with CSC210 lecture content to reinforce theoretical concepts through practical implementation.

Laboratory Learning Objectives

Upon successful completion of this lab course, students will be able to:

  • Design and simulate digital circuits from basic gates to complex CPU components using Logisim.
  • Implement and test arithmetic logic units, memory systems, and control units.
  • Write, debug, and optimize MIPS assembly language programs using the MARS simulator.
  • Build a complete 8-bit CPU capable of executing a custom instruction set.
  • Demonstrate understanding of the hardware-software interface through practical projects.
  • Use professional simulation tools for circuit design and assembly programming.

17-Week Lab Course Outline

Phase 1: Tool Mastery (Weeks 1-4)

Lab: Logisim basics, creating simple circuits
Exercise: Build basic gates, verify truth tables

Lab: Build decoders, encoders, and comparators
Exercise: Create 8-bit magnitude comparator

Deliverable: Working 8-bit ALU

Lab: Implement loops, conditionals, case structures

Exercise: Implement quicksort

Key Features of Lab-Lecture Integration

Progressive Checkpoints

Lab deliverables are completed just before corresponding lecture checkpoints, providing hands-on experience that reinforces theoretical understanding.

Scaffolded Learning

Each lab builds on previous work, creating a complete learning progression from basic components to a functional CPU.

Synchronized Timing

Lab activities are carefully timed to provide practical context for lecture content while preparing students for upcoming theoretical concepts.

Practical Validation

Students can test and validate theoretical concepts through hands-on implementation, deepening understanding of computer architecture principles.

Lab Policies & Student Support

Lab Assignments & Late Submissions

All lab deliverables are expected to be submitted as defined in the assignment description and NO LATER THAN the provided Due Date. For the lab midterm and final demonstration, NO LATE SUBMISSIONS will be accepted. For weekly lab deliverables, submissions will remain open however if submission is completed after we have reviewed the deliverable in class (generally will review first class after due date), a 15% reduction in your score will be applied. If you are having difficulties with circuit design or programming, please communicate with the instructor before the deadline.

Technical Requirements & Software Support

Logisim Evolution: Students must use the latest stable version of Logisim Evolution for all circuit design work. Technical support is available during office hours.

MARS Simulator: All MIPS programming must be completed using the MARS simulator. Students experiencing compatibility issues should contact the instructor immediately.

File Formats: All circuit files must be submitted in .circ format. Assembly programs must be submitted as .asm files with accompanying documentation.

Video Demonstrations: Final project demonstrations must be submitted as MP4 files with clear audio narration explaining the CPU functionality.

Academic Integrity in Lab Work

While collaboration is encouraged during lab sessions, all submitted work must be your own. Students may discuss concepts and troubleshoot together, but circuit designs and code implementations must be individual work. Sharing of .circ files or .asm source code constitutes academic misconduct. For specific guidelines please refer to Admin Procedure 5500.1 for the Student Conduct Guidelines and Admin Procedure 5500.2 on academic honesty.

Lab Safety & Best Practices

Digital Safety: Always save your work frequently and maintain backup copies of all circuit files and programs. Use cloud storage or external drives to prevent data loss.

Time Management: Lab projects build progressively - falling behind early in the semester makes later projects significantly more difficult. Plan adequate time for debugging and testing.

Resource Usage: Be mindful of computational resources when running complex simulations. Large circuits may require significant processing time.
