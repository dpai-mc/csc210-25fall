Midterm Exam - Part B: Building an 8-bit ALU

Objective: Build a complete 8-bit Arithmetic Logic Unit (ALU) in Logisim 2.7.1 that integrates all the circuit components from the course. Your ALU will perform seven operations controlled by a 3-bit opcode.

Submission: Upload your completed .circ file to Canvas.

Overview: What is an ALU?

An Arithmetic Logic Unit (ALU) is the computational core of a CPU. It performs arithmetic operations (like addition) and logical operations (like AND, OR, NOT) on binary data. Your 8-bit ALU will:

  • Accept two 8-bit inputs (A and B)
  • Accept a 3-bit control signal (opcode) to select the operation
  • Produce an 8-bit output based on the selected operation
  • Integrate all the circuits you've built throughout this unit

Step 2: ALU Opcode Specification

Your ALU uses a 3-bit opcode to select which operation to perform. The opcode mapping is as follows:

Opcode Operation Description
000 Adder Performs 8-bit addition: Output = A + B
001 Shift Right Shifts input A right by 1 bit
010 Shift Left Shifts input A left by 1 bit
011 NOT Bitwise NOT of input A: Output = ~A
100 AND Bitwise AND: Output = A AND B
101 OR Bitwise OR: Output = A OR B
110 XOR Bitwise XOR: Output = A XOR B
Note: Opcode 111 is unused. Your ALU design should handle this case appropriately (e.g., output all zeros or default to a specific operation).

Step 3: Integrate Circuits into the ALU

Integration Architecture

To build your complete ALU, you need to:

  1. Create Input Pins: Two 8-bit input pins (A and B) and one 3-bit input pin (Opcode)
  2. Connect All Circuit Components: Wire both inputs A and B to all circuits that need them
  3. Add a Multiplexer: Use an 8:1 multiplexer (8-bits wide) to select which circuit's output to send to the ALU output
  4. Connect the Multiplexer: Wire each circuit's output to the appropriate multiplexer input based on the opcode table above
  5. Connect Opcode to Selector: Wire the 3-bit opcode input to the multiplexer's selector input
  6. Create Output Pin: Create an 8-bit output pin connected to the multiplexer output
Tip: Use splitters in Logisim to separate multi-bit wires when needed. Keep your circuit organized with clear wire routing and labels for inputs and outputs.

Step 4: Test Your ALU

Thorough testing is essential to ensure your ALU works correctly. Use the test cases below to verify all operations and flags work properly.

Flag Definitions:
  • Is Even: Result's LSB (least significant bit) is 0
  • Is Larger: A > B (comparison of inputs)
  • Cout: Carry out flag (1 if addition overflows, or bit shifted out for shift operations)
  • Zero: Result is 00000000
